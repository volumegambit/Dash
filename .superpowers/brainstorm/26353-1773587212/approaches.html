Three Approaches to Reorganize AgentDetail

Click the one that feels closest to what you want. We can mix and match ideas too.

A

Tabbed Interface

Header + status always visible. Content split into Overview | Config | Skills | Logs tabs. Matches the pattern already used on the Messaging Apps page.

Pros

  • Near-zero scrolling per tab
  • Existing pattern in the codebase
  • Clear grouping — status vs config vs logs
  • Skills gets dedicated space

Cons

  • Can't see logs while editing config
  • Extra click to switch contexts
B

Collapsible Accordion

Keep single-page layout but make every section collapsible. Sticky header at top. Only status open by default, everything else collapsed with a 1-line summary.

Pros

  • Everything on one page
  • User controls what's visible
  • Minimal code changes

Cons

  • Still a long page when multiple sections open
  • Doesn't solve the "equal weight" problem deeply
  • Skills section still complex when expanded
C

Summary Cards + Drawer Editing

Compact dashboard of summary cards showing config at a glance (model name, tool count, prompt preview). Click a card to open a side drawer for editing. Logs in a toggleable bottom panel.

Pros

  • Entire page visible without scrolling
  • Edit mode doesn't disrupt layout
  • Most modern/polished feel

Cons

  • Most code to build (drawers, cards)
  • No existing drawer pattern in codebase
  • Mobile/narrow viewports need fallback

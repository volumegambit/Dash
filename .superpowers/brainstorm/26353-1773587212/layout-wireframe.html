What goes where

Area Contents
Header (always visible) Name, status badge, action buttons, uptime, workspace, channel health dots, connect banner, startup errors
Overview tab Runtime details (PID, ports, created date), channel connections list
Configuration tab Models (with edit), System Prompt (with edit), Tools (with edit) — each as a collapsible card
Skills tab Skills list, inline editor, discovery paths, remote URLs, create new skill
Logs tab Level filter buttons + full-height LogViewer

Key design decisions

  • Channel health dots in header — you see connection status at a glance without switching to Overview
  • Config sections as collapsible cards — within the Config tab, each section (models/prompt/tools) is a card that expands for editing
  • Skills gets its own tab — it's complex enough to warrant dedicated space rather than being crammed alongside models/tools
  • Logs tab uses full available height — no content above or below the log viewer to maximize visible lines
  • Startup errors show above tabs — critical errors are always visible, not hidden in a tab

# Rewrite & Polish

## When to use
- The user asks to "polish", "edit", "improve", "proofread", "tighten", "clean up", "reword", or "make this clearer".
- Someone shares an email, message, post, paragraph, or document and wants it to read better.
- The user wants a stronger version of their own writing without changing what it says.

Skip when the user wants new content written from scratch (that's drafting, not polishing) or a translation.

## Workflow
1. Confirm intent if unclear (ask in one message): the **goal** (clarity, brevity, formality, friendliness, persuasion?), the **audience**, and any **hard constraints** (length limit, must-keep phrases, terminology). If the user just says "polish this", assume clarity + concision while keeping their voice.
2. Read the original closely and identify what it is trying to say. Preserving meaning is the top priority — never add claims, change facts, or drop the author's intent.
3. Edit for, in order:
   - **Clarity** — fix ambiguous pronouns, vague subjects, tangled sentences; lead with the main point.
   - **Concision** — cut filler ("in order to" → "to"), redundancy, and hedging; prefer strong verbs over nominalizations.
   - **Tone** — match the requested register; keep it consistent throughout.
   - **Correctness** — fix grammar, spelling, punctuation, and agreement.
4. Preserve the author's voice and any domain terms or names — polish, don't rewrite into a different person's style unless asked.
5. Respect constraints: stay within any length/character limit (note the new count for emails, tweets, etc.).
6. Note the major changes you made and why — group them, don't annotate every comma. Keep rationale to the substantive edits (restructured the opening, cut a redundant paragraph, shifted to active voice).
7. If the text has a factual error or an internal contradiction, flag it rather than silently "fixing" it — you may be wrong about the intent.

## Output
1. **Polished version** — the revised text, ready to copy, in a quoted block or fenced block.
2. **What changed** — 2–5 bullets covering the major edits and the reason for each (e.g., "Tightened the first two sentences into one; the original buried the ask").
3. Optional: if a sentence has a genuine tradeoff (e.g., shorter vs. warmer), offer the alternative in one line.
4. Close with: "Want it more formal/casual, shorter, or in a different voice?"

## Guardrails
- Never change the meaning, add information, or remove substantive content the author intended to keep — clarify with a question if a passage is ambiguous.
- Do not over-edit into generic corporate prose; preserve personality, humor, and idiom unless the user wants them gone.
- Don't invent facts, statistics, or citations; flag anything that reads as a factual error instead of rewriting around it.
- Keep proper nouns, quotes, code, and technical terms exactly as given unless they contain a clear typo.
- If the original is already strong, say so and make only light touches rather than manufacturing changes.

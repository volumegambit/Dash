Get your first agent running in a few minutes. Dash runs entirely on your machine or private cloud — no data is sent anywhere except the LLM provider you choose.

## Prerequisites

- **Node.js 22+** — Dash uses ESM and modern language features requiring Node 22 or later
- **An LLM API key** — for example, sign up at [console.anthropic.com](https://console.anthropic.com) for an Anthropic key

## Installation

<Steps>
  <Step title="Clone and install">
    ```bash
    git clone <repo-url> Dash
    cd Dash
    npm install
    ```
  </Step>
  <Step title="Build all packages">
    ```bash
    npm run build
    ```
  </Step>
</Steps>

## Configuration

Set up your credentials using either environment variables or a credentials file.

<Tabs>
  <Tab title="Environment variables">
    Copy the example `.env` file and fill in your key:

    ```bash
    cp .env.example .env
    ```

    Edit `.env`:

    ```env
    ANTHROPIC_API_KEY=sk-ant-...
    LOG_LEVEL=info
    ```
  </Tab>
  <Tab title="Credentials file">
    Copy the example credentials file:

    ```bash
    cp config.example/credentials.json config/credentials.json
    ```

    Edit `config/credentials.json`:

    ```json
    {
      "anthropic": {
        "apiKey": "sk-ant-..."
      }
    }
    ```
  </Tab>
</Tabs>

<Note>
  Environment variables take precedence over `credentials.json` when both are set.
</Note>

### Agent configuration

Copy the example config:

```bash
cp config.example/dash.json config/dash.json
```

See [configuration](/configuration) for the full schema.

## Running Dash

<Tabs>
  <Tab title="TUI (terminal)">
    The TUI provides an interactive REPL that connects directly to an agent:

    ```bash
    npm run tui
    ```

    You'll see a header showing the active agent, model, tools, and workspace path. Type a message and press Enter.
  </Tab>
  <Tab title="Agent server">
    The agent server starts a headless server with a Management API (HTTP) and Chat API (WebSocket):

    ```bash
    npm run dev
    ```

    To enable the APIs, set tokens in your `.env`:

    ```env
    MANAGEMENT_API_TOKEN=your-management-token
    CHAT_API_TOKEN=your-chat-token
    ```

    The Management API listens on port 9100 and the Chat API on port 9101. Connect via WebSocket to interact with agents.
  </Tab>
  <Tab title="Docker">
    ```bash
    docker compose up --build
    ```

    Session data is persisted via a volume mount at `./data/sessions`. Config is mounted read-only from `./config`.
  </Tab>
</Tabs>

Get your first AI agent teammate up and running in minutes. No coding required.

## Prerequisites

- **Node.js 22+** — Dash uses ESM and modern language features requiring Node 22 or later
- **An LLM API key** — for example, sign up at [console.anthropic.com](https://console.anthropic.com) for an Anthropic key

## Getting Your Anthropic API Key

Dash works with Claude from Anthropic out of the box. To use it, you need an Anthropic account and an API key.

**Step 1:** Go to [console.anthropic.com](https://console.anthropic.com) and create an account.

**Step 2:** Once logged in, click **API Keys** in the left sidebar.

**Step 3:** Click **Create Key**, give it a name (e.g. "Dash"), and copy the key — it starts with `sk-ant-`.

<Note>
  **Keep your API key private.** Never share it or commit it to version control. Dash stores it encrypted on your machine.
</Note>

**Billing:** Anthropic charges per token used. Running agents occasionally costs a few cents per session. New accounts receive free credits to get started.

## Installation

<Steps>
  <Step title="Clone and install">
    ```bash
    git clone <repo-url> Dash
    cd Dash
    npm install
    ```
  </Step>
  <Step title="Build all packages">
    ```bash
    npm run build
    ```
  </Step>
</Steps>

## Launch Mission Control

Mission Control is the recommended way to manage your agents. It gives you a desktop app where you can deploy agents, chat with them, and connect them to messaging platforms.

```bash
npm run mc:dev
```

The app opens automatically. From here, you will connect an AI provider and deploy your first agent.

## Connect your AI provider

Before deploying an agent, you need to connect at least one AI provider. Mission Control stores your API key in an encrypted file on your machine.

<Steps>
  <Step title="Open Connections">
    In the Mission Control sidebar, click **Connections**.
  </Step>
  <Step title="Add your API key">
    Select your provider (Anthropic, OpenAI, or Google) and paste your API key. If this is your first time, you will be asked to create an encryption password. This password protects all your stored secrets.
  </Step>
  <Step title="Verify">
    Once saved, the provider shows a connected status. All agents that use this provider will share the key automatically.
  </Step>
</Steps>

Dash also supports OpenAI and Google models. See [AI Providers](/ai-providers) for details on connecting additional providers.

## Deploy your first agent

Now that your provider is connected, you can deploy an agent.

<Steps>
  <Step title="Open the Deploy wizard">
    Click **Deploy** in the sidebar. This opens the deployment wizard.
  </Step>
  <Step title="Choose a name">
    Give your agent a descriptive name, like "assistant" or "code-reviewer".
  </Step>
  <Step title="Select a model">
    Pick the model your agent will use. If you are not sure, Claude Sonnet 4 is a great starting point — it balances speed and capability.
  </Step>
  <Step title="Write a system prompt">
    Tell your agent who it is and how to behave. For example:

    ```
    You are a helpful assistant. You help with research, writing,
    and general tasks. Be concise but thorough.
    ```
  </Step>
  <Step title="Pick tools">
    Select which tools your agent can use. A good starter set is **bash**, **read**, and **write**. See [Tools](/tools) for the full list.
  </Step>
  <Step title="Deploy">
    Click **Deploy**. Mission Control starts your agent as a background process with its own workspace and ports.
  </Step>
</Steps>

Your agent appears in the sidebar. Click it to check that the status shows "running".

## Chat with your agent

Click **Chat** in the sidebar, select your agent, and type a message. You will see the agent thinking, using tools, and responding in real time.

That is it — your first agent is deployed and ready to work.

## Other ways to run Dash

Mission Control is the recommended approach, but Dash also supports other interfaces for specific use cases.

- **TUI** — a terminal interface for quick interactions: `npm run tui`
- **Agent Server** — run agents headlessly for automation: `npm run dev`
- **CLI** — manage deployments from the terminal: `npm run mc-cli -- <command>`

See [Mission Control](/mission-control) and [CLI Reference](/cli-reference) for details.

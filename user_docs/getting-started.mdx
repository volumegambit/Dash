Get your first AI agent teammate up and running in minutes. No coding required.

## Prerequisites

- **Node.js 22+** — Dash uses ESM and modern language features requiring Node 22 or later
- **An LLM API key** — for example, sign up at [console.anthropic.com](https://console.anthropic.com) for an Anthropic key

## Getting Your Anthropic API Key

Dash uses the Claude AI model from Anthropic. To use it, you need an Anthropic account and an API key.

**Step 1:** Go to [console.anthropic.com](https://console.anthropic.com) and create an account.

**Step 2:** Once logged in, click **API Keys** in the left sidebar.

**Step 3:** Click **Create Key**, give it a name (e.g. "Dash"), and copy the key — it starts with `sk-ant-`.

<Note>
  **Keep your API key private.** Never share it or commit it to version control. Dash stores it encrypted on your machine.
</Note>

**Billing:** Anthropic charges per token used. Running agents occasionally costs a few cents per session. New accounts receive free credits to get started.

<Steps>
  <Step title="Clone and install">

    ```bash
    npm install
    ```
  </Step>
  <Step title="Build all packages">

## Set up credentials

Your agents need access to an AI provider. Set that up once and all your agents share it.

<Tabs>
  <Tab title="Mission Control (recommended)">
    When using Mission Control (desktop app or CLI), secrets are stored in an encrypted file at `~/.mission-control/secrets.enc`. The first time you run a command, you'll be prompted to create an encryption password:

    ```bash
    npm run mc-cli -- secrets set anthropic-api-key
    # Create encryption password: ********
    # Confirm password: ********
    # Value for 'anthropic-api-key': sk-ant-...
    # Secret 'anthropic-api-key' stored.
    ```

    Your password-derived key is cached in the OS keychain (macOS Keychain / Linux secret-tool), so subsequent commands don't prompt again. Use `mc lock` to clear the cache.
  </Tab>
  <Tab title="Environment variables">
    For the agent server and TUI, set your key in a `.env` file:

    ```bash
    cp .env.example .env
    ```

    Edit `.env`:

    ```env
    ANTHROPIC_API_KEY=sk-ant-...
    LOG_LEVEL=info
    ```
  </Tab>
  <Tab title="Credentials file">
    Copy the example credentials file:

    ```bash
    cp config.example/credentials.json config/credentials.json
    ```

    Edit `config/credentials.json`:

    ```json
    {
      "anthropic": {
        "apiKey": "sk-ant-..."
      }
    }
    ```
  </Tab>
</Tabs>

<Note>
  For the agent server: environment variables take precedence over `credentials.json`. For Mission Control deployments: the encrypted secret store is the primary source, and secrets are injected into agents via temporary read-then-delete files.
</Note>

## Launch your team

Choose how you want to manage your agent team.

<Tabs>
  <Tab title="Mission Control (recommended)">
    Use Mission Control to deploy and chat with your agents from a desktop app. Recommended for most users.

    ```bash
    npm run mc:dev
    ```
  </Tab>
  <Tab title="TUI">
    A terminal interface for managing and chatting with agents directly. Great for quick interactions.

    ```bash
    npm run tui
    ```

    You'll see a header showing the active agent, model, tools, and workspace path. Type a message and press Enter.
  </Tab>
  <Tab title="Agent Server">
    Run the agent server headlessly — ideal for automated workflows or server deployments.

    ```bash
    npm run dev
    ```

    To enable the APIs, set tokens in your `.env`:

    ```env
    MANAGEMENT_API_TOKEN=your-management-token
    CHAT_API_TOKEN=your-chat-token
    ```

    The Management API listens on port 9100 and the Chat API on port 9101. Connect via WebSocket to interact with agents.
  </Tab>
  <Tab title="Docker">
    Deploy your agent team in a container. Useful for consistent, portable setups.

    ```bash
    docker compose up --build
    ```

    Session data is persisted via a volume mount at `./data/sessions`. Config is mounted read-only from `./config`.
  </Tab>
</Tabs>
